ИНСТРУКЦИЯ ПО ЗАПИСИ ВИДЕО
ДЕМОНСТРАЦИЯ СИСТЕМЫ УЧЕТА ЗАЯВОК НА РЕМОНТ КЛИМАТИЧЕСКОГО ОБОРУДОВАНИЯ

1. ПОДГОТОВКА К ЗАПИСИ

1.1 Техническая подготовка
- Убедитесь, что система запущена: npm start
- Откройте браузер на http://localhost:3001
- Подготовьте программу для записи экрана (OBS Studio, Bandicam, или встроенную в Windows)
- Настройте качество записи: 1920x1080, 30fps
- Проверьте звук микрофона

1.2 Подготовка контента
- Подготовьте план демонстрации
- Закройте лишние программы и вкладки
- Очистите рабочий стол от лишних файлов
- Подготовьте тестовые данные для демонстрации

2. СТРУКТУРА ВИДЕО ПО ЗАДАНИЮ 1 (5-7 минут)

2.1 Введение (30 секунд)
"Добро пожаловать! Меня зовут [Ваше имя]. Сегодня я продемонстрирую систему учета заявок на ремонт климатического оборудования, разработанную в рамках задания 1."

2.2 Обзор алгоритмов (2 минуты)
- Откройте файл docs/algorithms_and_flowcharts.md
- Покажите блок-схемы основных алгоритмов:
  * Алгоритм аутентификации
  * Алгоритм назначения заявок
  * Алгоритм расчета статистики
- Объясните каждую блок-схему кратко

2.3 Демонстрация работы алгоритмов (3 минуты)
- Войдите в систему как менеджер (login1/pass1)
- Покажите создание новой заявки
- Продемонстрируйте назначение специалиста
- Откройте раздел "Отчеты" и покажите расчет статистики
- Объясните, как работают алгоритмы на практике

2.4 Заключение по заданию 1 (30 секунд)
"Все алгоритмы работают корректно, блок-схемы соответствуют ГОСТ 19.701, система готова к использованию."

3. СТРУКТУРА ВИДЕО ПО ЗАДАНИЮ 3 (5-7 минут)

3.1 Введение (30 секунд)
"Теперь продемонстрирую выполнение задания 3 - руководство системного программиста и анализ качества кода."

3.2 Демонстрация руководства (2 минуты)
- Откройте docs/system_programmer_guide.txt
- Покажите структуру документа
- Пролистайте основные разделы:
  * Архитектура системы
  * Установка и настройка
  * Структура базы данных
  * Устранение неполадок
- Объясните соответствие стандартам ЕСПД

3.3 Анализ качества кода (2.5 минуты)
- Откройте docs/code_quality_report.txt
- Покажите результаты анализа:
  * Соблюдение стандартов кодирования
  * Безопасность кода
  * Тестирование
  * Метрики качества
- Откройте несколько файлов кода для демонстрации:
  * services/statisticsService.js (показать JSDoc комментарии)
  * middleware/roleAccess.js (показать обработку ошибок)
  * tests/statisticsService.test.js (показать property-based тесты)

3.4 Демонстрация тестов (1.5 минуты)
- Откройте терминал
- Запустите: npm test
- Покажите результаты тестирования
- Объясните property-based тестирование

3.5 Заключение по заданию 3 (30 секунд)
"Руководство создано в соответствии с ЕСПД, качество кода соответствует современным стандартам, система готова к промышленной эксплуатации."

4. СОВЕТЫ ПО ЗАПИСИ

4.1 Технические советы
- Говорите четко и не торопясь
- Делайте паузы между разделами
- Используйте указатель мыши для выделения важных элементов
- Увеличивайте шрифт в редакторе для лучшей видимости

4.2 Содержательные советы
- Объясняйте, что показываете на экране
- Связывайте демонстрацию с теоретическими знаниями
- Подчеркивайте соответствие стандартам и требованиям
- Показывайте практическую ценность разработанного решения

4.3 Структурные советы
- Следуйте подготовленному плану
- Не отвлекайтесь на детали
- Укладывайтесь в отведенное время
- Делайте четкие переходы между разделами

5.1 Начало видео
"Здравствуйте! Меня зовут [Ваше имя]. Сегодня я продемонстрирую выполнение заданий 1 и 3 курсового проекта - систему учета заявок на ремонт климатического оборудования."

5.2 Переход к заданию 1
"Начнем с задания 1 - разработки алгоритмов и блок-схем. Все алгоритмы оформлены согласно ГОСТ 19.701 и реализованы в рабочей системе."

5.3 Переход к заданию 3
"Теперь перейдем к заданию 3. Я создал руководство системного программиста в соответствии с требованиями ЕСПД и провел анализ качества кода."

5.4 Завершение видео
"Спасибо за внимание! Все задания выполнены в полном объеме, система готова к использованию. Документация и исходный код доступны в репозитории."

7. ПОСЛЕ ЗАПИСИ

7.1 Обработка видео
- Обрежьте лишние части в начале и конце
- Добавьте титульный слайд с названием проекта
- Проверьте качество звука и изображения
- Сохраните в формате MP4

7.2 Финальная проверка
- Просмотрите видео полностью
- Убедитесь, что все ключевые моменты показаны
- Проверьте, что звук синхронизирован с изображением
- Убедитесь, что видео укладывается в требуемое время

8. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ К ВИДЕО

- Формат: MP4
- Разрешение: 1920x1080 (Full HD)
- Частота кадров: 30fps
- Качество звука: 44.1kHz, стерео
- Продолжительность: 10-15 минут общая
- Размер файла: не более 500MB
